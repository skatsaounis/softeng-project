Παράδειγμα Περίπτωσης Χρήσης: Καταχώριση Φραγής

  • Δράστες: φράσσων, φρασσόμενος, client φράσσοντα, blocking server, βάση δεδομένων

  • Προαπαιτούμενα (Preconditions):

    • Ο φράσσων είναι ενεργός.
    • Ο blocking server είναι ενεργός.

  • Βασική Πορεία Χρήσης:

    1. O φράσσων πληκτρολογεί το όνομα χρήστη του φρασσόμενου και επιλεγει τη φραγή.
    2. Ο blocking server λαμβάνει το αίτημα για φραγή από τον client του φράσσοντα.
    3. Ο blocking server αναζητεί στη βάση δεδομένων χρήστη με το όνομα χρήστη του φρασσόμενου.
    4. Ο blocking server επιβεβαιώνει ότι ο φρασσόμενος δεν βρίσκεται στις φραγμένες επαφές του φράσσοντα, αναζητώντας τη βάση δεδομένων.
    5. Ο blocking server ενημερώνει τη βάση δεδομένων, προσθέτοντας τον φρασσόμενο στις φραγμένες επαφές του φράσσοντα.
    6. Ο blocking server αποστέλει επιβεβαίωση επιτυχούς φραγής στον client του φράσσοντα.
    7. Ο client του φράσσοντα εμφανίζει κατάλληλο μύνημα επιτυχίας.

  • Εναλλακτικές πορείες χρήσης:

    • Εάν δεν υπάρχει χρήστης με το όνομα χρήστη του φρασσόμενου στη βάση δεδομένων:

      • Ο blocking server παράγει μύνημα αποτυχίας φραγής.
      • Ο blocking server αποστέλει το μήνυμα στον client του φράσσοντα.
      • Ο client του φράσσοντα εμφανίζει το σχετικό μύνημα.

    • Εάν ο φρασσόμενος βρίσκεται ήδη στις φραγμένες επαφές του φράσσοντα:

      • Ο blocking server αποστέλει επιβεβαίωση επιτυχούς φραγής στον client του φράσσοντα.
      • Ο client του φράσσοντα εμφανίζει κατάλληλο μύνημα επιτυχίας.
