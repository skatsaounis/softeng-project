Standard call

User B is online.
User A attempts to call user B.
Call succeeds; Users A and B communicate with each other.

Standard call - Busy Callee

User B is online and communicates with user C.
User A attempts to call user B.
Call fails; User A is notified that user B is not available - busy.

Standard call - Callee rejects call

User B is online.
User A attempts to call user B.
User B rejects the call. Call fails; User A is notified that user B is not available - busy.

Call to offline user

User B is offline. User A attempts to call user B.
Call fails; User A is notified that user B is offline.

Self Call

User A calls themselves.
Call fails; User A is notified that user A is not available - busy.

Blocking scenario

User B has blocked user A. User B is either online or offline.
User A attempts to call user B.
Call fails; User A is notified that user B is not available - busy.

Blocking Registration scenario

User A either has or has not registered blocking from address B.
User A attempts to register blocking from address B.
Blocking succeeds; From now on, user A has registered blocking from address B.

Blocking Removal scenario

User A either has or has not registered blocking from address B.
User A attempts to remove blocking from address B.
Blocking removal succeeds; From now on, user A has removed blocking from address B.

Blocker calls blocked user

User A has blocked user B. User B is online.
User A attempts to call user B.
Call succeeds; Users A and B communicate with each other.

Forwarding scenario

User B has registered forwarding to address C.
Users B or C are either online or offline.
User A attempts to call user B.
Falls to one of Standard Call Scenarios amongst users A and C.

Forwarding Registration scenario

User A has not registered forwarding.
User A attempts to register forwarding to address B.
Forwarding succeeds; From now on, user A has registered forwarding to address B.

Forwarding Registration scenario - ReForwarding

User A has registered forwarding to address B.
User A attempts to register forwarding to address C too.
Forwarding succeeds; From now on, user A has registered forwarding to address C.

Forwarding Registration, Chain scenario - forwarding cycle

User A has registered forwarding to address B.
User B has registered forwarding to address C.
User C has registered forwarding to address D.
User D attempts to register forwarding to address A, B or C.
Forwarding fails; User D is notified that forwarding cannot succeed due to forwarding cycle.

Forwarding Removal scenario

User A attempts to remove forwarding.
Forwarding removal succeeds; From now on, user A does not forward calls.

Forwarding Scenario - forwardee blocks caller

User B has registered forwarding to address C.
User C has registered forwarding to address D.
User D has registered forwarding to address E.
User A attempts to call User B.
Someone from Users C, D or E blocks User A.
Call fails; User A is notified that user B is not available - busy.

Forwarding Scenario - forwarder blocks caller

User B has registered forwarding to address C.
User A attempts to call User B.
User B has blocked User A.
Call fails; User A is notified that user B is not available - busy.

Forwarding Scenario - forwardee blocks forwarder

User A has registered forwarding to address B.
User B has registered forwarding to address C.
User C has registered forwarding to address D.
User E attempts to call D.
Someone from forwardees blocks someone amongst previous forwarders.
Call fails; User E is notified that user D is not available - busy.

Call to Forwarding Chain

User A has registered forwarding to address B.
User B has registered forwarding to address C.
User C has registered forwarding to address D.
User D attempts to call A, B or C.
Falls to Self call Scenario.

User Selects a Program

User wants to select program_A from available programs.
User has not added a program the last (5 minutes).
From now on, program_A is applied to user's calls.

User Selects a Program before expiration of a previous program.

User wants to select program_A from available programs.
User has added program_B (3 minutes) ago.
User is notified that they cannot change program_B yet.

Basic call charge recording

Caller and callee start to communicate.
Caller or callee decides to end call.
Caller is charged according to their billing program.

Total charge request

User A requests to view the total charge they are required to pay.
User is notified of their total charge.
